Introduction

This Privacy Policy ("Policy") describes how Mr.Memo (the "App") handles user information.

Mr.Memo is designed with user privacy as the highest priority, and as a general principle, the App does not collect any personal information.

1. Information We Handle

1.1 Information We Do NOT Collect

The App does not collect any of the following:

  • Personal information such as name, address, or phone number
  • Email addresses or user account information
  • Location data
  • Contacts or photo gallery
  • Device identifiers (advertising ID, etc.)
  • Usage logs or analytics data
  • Crash reports

1.2 Information Stored Locally on Your Device

The App stores the following user-entered data only on your device. This data is never transmitted to external servers, and no third party (including the developer) can access it:

  • Memo content
  • Folders, tags, and favorites
  • App settings (theme, font size, language, etc.)

2. Data Storage and Management

2.1 Storage Location

All data created within the App is stored in a local database (SQLite) on your device.

This data is stored within Android's "app-specific storage", and by Android's standard design, it is inaccessible to other apps (except on rooted devices). When you uninstall the App, this database is completely deleted.

2.2 Data Transmission

The App does not transmit user data to any external server for any purpose.

2.3 Network Communication

The App itself does not engage in any communication with external servers (HTTP, HTTPS, WebSocket, etc.).

If the user chooses to share memo content with other apps (such as SNS apps or email apps), the data is passed to those apps through Android's standard system functions. In such cases, how the receiving app handles the data is governed by that app's privacy policy.

3. About the Backup Feature

The App includes a feature to export memos, folders, and other data as encrypted backup files, and to restore data from those files.

3.1 Encryption Method

Backup files are protected using industry-standard encryption technologies:

  • Encryption Algorithm: AES-256 (CBC mode)
  • Key Derivation Function: PBKDF2-SHA256 (150,000 iterations)
  • Tamper Detection: HMAC-SHA256

Without the password set by the user, no one can decrypt the contents of a backup file. No one — including the developer — can know your password.

3.2 Password Management

Important: If you forget your backup password, the data cannot be recovered by any means. Please store your password securely.

3.3 Backup File Location

Backup files are saved to a location you choose (device storage, cloud storage, etc.). The App will never transmit backup files to any location unless you explicitly choose to do so.

4. Third-Party Disclosure

The App does not provide user data to any third party.

The App contains no third-party services such as:

  • Advertising services (e.g., Google AdMob)
  • Analytics services (e.g., Google Analytics, Firebase Analytics)
  • Crash reporting services (e.g., Sentry, Crashlytics)
  • Social network integrations

5. Information About Children

The App is not targeted at any specific age group. However, in consideration of children's privacy, the App does not collect any personal information from users under the age of 13.

6. Data Deletion

By uninstalling the App, you can delete all data that the App has stored on your device.

No separate data deletion request is necessary. The developer does not retain any user data, so there is no information held externally that needs to be deleted.

7. Device Permissions

The App uses the following Android permission:

  • Internet permission (android.permission.INTERNET): The App itself does not communicate with any external server. This permission is used to invoke the system standard sharing feature when the user shares memo content with other apps (such as SNS apps) via the "Open with another app" function.

The App does not request any of the following permissions: storage, contacts, camera, microphone, location, calendar, or notifications.

Backup files are read and written through Android's standard Storage Access Framework (SAF) document picker, which only accesses files and folders explicitly selected by the user. No storage permission is required for the App.

8. Security Measures

The App implements the following measures to protect user data:

  • Strong encryption of backup files (AES-256 + PBKDF2-SHA256)
  • Password strength validation (12+ characters, mix of letters, numbers, and symbols)
  • Tamper detection (HMAC-SHA256)
  • Concealment of debug information in production builds

However, no electronic system can be guaranteed to be 100% secure. Users assume responsibility for the use of this App.

We recommend reviewing this Policy periodically.
