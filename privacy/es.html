Introducción

Esta Política de Privacidad ("Política") describe cómo Mr.Memo (la "Aplicación") maneja la información del usuario.

Mr.Memo está diseñada con la privacidad del usuario como máxima prioridad y, como principio general, la Aplicación no recopila ninguna información personal.

1. Información que Manejamos

1.1 Información que NO Recopilamos

La Aplicación no recopila ninguna de la siguiente información:

  • Información personal como nombre, dirección o número de teléfono
  • Direcciones de correo electrónico o información de cuenta de usuario
  • Datos de ubicación
  • Contactos o álbumes de fotos
  • Identificadores de dispositivo (ID de publicidad, etc.)
  • Registros de uso o datos de análisis
  • Informes de fallos

1.2 Información Almacenada Solo Localmente en su Dispositivo

La Aplicación almacena los siguientes datos ingresados por el usuario solo en su dispositivo. Estos datos nunca se transmiten a servidores externos y son inaccesibles para terceros, incluyendo al desarrollador:

  • Contenido de las notas
  • Carpetas, etiquetas y favoritos
  • Configuración de la aplicación (tema, tamaño de fuente, idioma, etc.)

2. Almacenamiento y Gestión de Datos

2.1 Ubicación de Almacenamiento

Todos los datos creados en la Aplicación se almacenan en una base de datos local (SQLite) en su dispositivo.

Estos datos se almacenan en el "almacenamiento específico de la aplicación" de Android y son inaccesibles para otras aplicaciones según el diseño estándar de Android (excepto en dispositivos con permisos de root). La base de datos se elimina por completo al desinstalar la Aplicación.

2.2 Transmisión de Datos

La Aplicación no transmite datos del usuario a ningún servidor externo para ningún propósito.

2.3 Comunicación de Red

La Aplicación en sí no se comunica con ningún servidor externo (HTTP, HTTPS, WebSocket, etc.).

Si el usuario elige compartir el contenido de las notas con otras aplicaciones (como aplicaciones de redes sociales o aplicaciones de correo electrónico), los datos se entregan a esas aplicaciones a través de las funciones estándar del sistema Android. En este caso, cómo manejan los datos las aplicaciones receptoras se determina por las políticas de privacidad de esas aplicaciones.

3. Acerca de la Función de Copia de Seguridad

La Aplicación incluye una función para exportar notas, carpetas y otros datos como archivos de copia de seguridad cifrados, así como para restaurar datos de dichos archivos.

3.1 Método de Cifrado

Los archivos de copia de seguridad están protegidos por tecnología de cifrado estándar de la industria:

  • Algoritmo de cifrado: AES-256 (modo CBC)
  • Función de derivación de clave: PBKDF2-SHA256 (15,000 iteraciones)
  • Detección de manipulación: HMAC-SHA256

Sin la contraseña establecida por el usuario, nadie puede descifrar el contenido del archivo de copia de seguridad. Nadie, incluyendo al desarrollador, puede conocer su contraseña.

3.2 Gestión de Contraseñas

Importante: Si olvida su contraseña de copia de seguridad, los datos no podrán restaurarse de ninguna manera. Por favor, conserve su contraseña en un lugar seguro.

3.3 Ubicación del Archivo de Copia de Seguridad

Los archivos de copia de seguridad se guardan en la ubicación que elija (almacenamiento del dispositivo, almacenamiento en la nube, etc.). La Aplicación nunca transmite archivos de copia de seguridad a ninguna ubicación a menos que usted lo elija explícitamente.

4. Divulgación a Terceros

La Aplicación no proporciona datos del usuario a ningún tercero.

La Aplicación no incluye ninguno de los siguientes servicios de terceros:

  • Servicios de publicidad (como Google AdMob)
  • Servicios de análisis (como Google Analytics, Firebase Analytics)
  • Servicios de informes de fallos (como Sentry, Crashlytics)
  • Integración con redes sociales

5. Información Sobre Niños

La Aplicación no está dirigida a ningún grupo de edad en particular. Sin embargo, en consideración a la privacidad de los niños, la Aplicación no recopila ninguna información personal de usuarios menores de 13 años.

6. Eliminación de Datos

Al desinstalar la Aplicación, puede eliminar todos los datos almacenados por la Aplicación en su dispositivo.

No se requieren solicitudes separadas de eliminación de datos. El desarrollador no retiene ningún dato del usuario, por lo que no existe información a eliminar externamente.

7. Permisos del Dispositivo

La Aplicación utiliza los siguientes permisos de Android:

  • Permiso de Internet (android.permission.INTERNET): La Aplicación en sí no se comunica con ningún servidor externo. Este permiso se utiliza para invocar la función estándar de compartir del sistema cuando el usuario comparte contenido de notas con otras aplicaciones (como aplicaciones de redes sociales) a través de la función "Abrir con otra aplicación".

La Aplicación no solicita ninguno de los siguientes permisos: almacenamiento, contactos, cámara, micrófono, ubicación, calendario o notificaciones.

Los archivos de copia de seguridad se leen y escriben a través del Marco de Acceso al Almacenamiento (SAF) estándar de Android mediante el selector de documentos, que solo accede a archivos y carpetas explícitamente seleccionados por el usuario. La Aplicación no requiere permisos de almacenamiento.

8. Medidas de Seguridad

La Aplicación implementa las siguientes medidas para proteger los datos del usuario:

  • Cifrado fuerte de archivos de copia de seguridad (AES-256 + PBKDF2-SHA256)
  • Validación de fuerza de contraseña (12 o más caracteres, mezcla de letras, números y símbolos)
  • Detección de manipulación (HMAC-SHA256)
  • Información de depuración oculta en compilaciones de producción

Sin embargo, ningún sistema electrónico puede garantizar el 100% de seguridad. Los usuarios utilizan la Aplicación bajo su propia responsabilidad.

9. Cambios a Esta Política

Esta Política puede actualizarse debido a cambios en las leyes o adiciones de funciones. La "Última actualización" se actualizará en caso de cambios significativos.

Le recomendamos que revise esta Política periódicamente.
