소개

본 개인정보처리방침(이하 "본 방침")은 Mr.Memo(이하 "본 앱")가 사용자 정보를 어떻게 처리하는지에 대해 규정합니다.

본 앱은 사용자의 개인정보 보호를 최우선으로 하여 설계되었으며, 원칙적으로 어떠한 개인정보도 수집하지 않습니다.

1. 처리하는 정보

1.1 수집하지 않는 정보

본 앱은 다음 정보를 일체 수집하지 않습니다:

  • 이름, 주소, 전화번호 등 개인정보
  • 이메일 주소 또는 사용자 계정 정보
  • 위치 데이터
  • 연락처 또는 사진 앨범
  • 기기 식별자(광고 ID 등)
  • 사용 로그 또는 분석 데이터
  • 충돌 보고서

1.2 기기 내에만 저장되는 정보

본 앱은 사용자가 입력한 다음 데이터를 기기 내에만 저장합니다. 이러한 데이터는 외부 서버에 전송되지 않으며, 개발자를 포함한 어떠한 제3자도 접근할 수 없습니다:

  • 메모 내용
  • 폴더, 태그 및 즐겨찾기
  • 앱 설정(테마, 글꼴 크기, 언어 등)

2. 데이터 저장 및 관리

2.1 저장 위치

본 앱에서 작성된 모든 데이터는 사용자의 기기 내 로컬 데이터베이스(SQLite)에 저장됩니다.

이러한 데이터는 Android의 "앱 전용 저장소"에 저장되며, Android의 표준 설계상 다른 앱에서는 접근할 수 없습니다(루트 권한이 있는 기기 제외). 앱을 제거하면 해당 데이터베이스는 완전히 삭제됩니다.

2.2 데이터 전송

본 앱은 어떠한 목적으로도 사용자 데이터를 외부 서버에 전송하지 않습니다.

2.3 네트워크 통신

본 앱 자체는 외부 서버와 일체 통신하지 않습니다(HTTP, HTTPS, WebSocket 등).

사용자가 메모 내용을 다른 앱(SNS 앱, 이메일 앱 등)에 공유하기로 선택한 경우, 데이터는 Android의 표준 시스템 기능을 통해 해당 앱에 전달됩니다. 이 경우 수신 앱이 데이터를 어떻게 처리하는지는 해당 앱의 개인정보처리방침에 따라 결정됩니다.

3. 백업 기능에 대하여

본 앱은 메모, 폴더 및 기타 데이터를 암호화된 백업 파일로 내보내고 해당 파일에서 데이터를 복원하는 기능을 포함합니다.

3.1 암호화 방법

백업 파일은 업계 표준 암호화 기술로 보호됩니다:

  • 암호화 알고리즘: AES-256(CBC 모드)
  • 키 유도 함수: PBKDF2-SHA256(15,000회 반복)
  • 변조 감지: HMAC-SHA256

사용자가 설정한 비밀번호 없이는 누구도 백업 파일의 내용을 복호화할 수 없습니다. 개발자를 포함한 어느 누구도 사용자의 비밀번호를 알 수 없습니다.

3.2 비밀번호 관리

중요: 백업 비밀번호를 잊어버린 경우 데이터를 어떠한 방법으로도 복원할 수 없습니다. 비밀번호를 안전하게 보관해 주세요.

3.3 백업 파일 위치

백업 파일은 사용자가 선택한 위치(기기 저장소, 클라우드 저장소 등)에 저장됩니다. 본 앱은 사용자가 명시적으로 선택하지 않는 한 어떠한 위치에도 백업 파일을 전송하지 않습니다.

4. 제3자 공개

본 앱은 사용자 데이터를 어떠한 제3자에게도 제공하지 않습니다.

본 앱은 다음과 같은 제3자 서비스를 일체 포함하지 않습니다:

  • 광고 서비스(Google AdMob 등)
  • 분석 서비스(Google Analytics, Firebase Analytics 등)
  • 충돌 보고 서비스(Sentry, Crashlytics 등)
  • 소셜 네트워크 통합

5. 어린이에 관한 정보

본 앱은 특정 연령대를 대상으로 하지 않습니다. 그러나 어린이의 개인정보 보호를 고려하여 본 앱은 13세 미만 사용자로부터 개인정보를 수집하지 않습니다.

6. 데이터 삭제

본 앱을 제거함으로써 본 앱이 기기에 저장한 모든 데이터를 삭제할 수 있습니다.

별도의 데이터 삭제 요청은 필요하지 않습니다. 개발자는 어떠한 사용자 데이터도 보유하지 않으므로 외부에 삭제할 정보가 존재하지 않습니다.

7. 기기 권한

본 앱은 다음 Android 권한을 사용합니다:

  • 인터넷 권한(android.permission.INTERNET): 본 앱 자체는 외부 서버와 통신하지 않습니다. 이 권한은 사용자가 "다른 앱으로 열기" 기능을 통해 메모 내용을 다른 앱(SNS 앱 등)에 공유할 때 시스템 표준 공유 기능을 호출하기 위해 사용됩니다.

본 앱은 다음 권한을 일체 요청하지 않습니다: 저장소, 연락처, 카메라, 마이크, 위치, 캘린더, 알림.

백업 파일은 Android의 표준 저장소 액세스 프레임워크(SAF) 문서 선택기를 통해 읽고 쓰며, 이는 사용자가 명시적으로 선택한 파일과 폴더에만 액세스합니다. 본 앱은 저장소 권한을 필요로 하지 않습니다.

8. 보안 조치

본 앱은 사용자 데이터를 보호하기 위해 다음 조치를 구현합니다:

  • 백업 파일의 강력한 암호화(AES-256 + PBKDF2-SHA256)
  • 비밀번호 강도 검증(12자 이상, 문자, 숫자, 기호 혼합)
  • 변조 감지(HMAC-SHA256)
  • 프로덕션 빌드에서 디버그 정보 숨김

그러나 어떠한 전자 시스템도 100% 보안을 보장할 수 없습니다. 사용자는 자신의 책임 하에 본 앱을 사용합니다.
